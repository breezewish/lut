## Anti-patterns verdict

**LLM assessment:** The interface does not read as AI-generated. It uses a restrained rose/neutral product palette, flat hierarchy, familiar controls, and an image-first comparison surface without decorative cards, glass, gradients, ornamental motion, or filler copy.

**Deterministic scan:** Both the layout-only and complete `web/src/App.tsx` scans returned `[]`; the arbitrary Tailwind-value and z-index search returned no matches. Browser detection reported only the intentional single Inter family and compact 12–16 px product scale. Those choices conform to the product register rather than indicating a weak hierarchy.

**Visual overlays:** Mutable detector injection succeeded in headless Chromium for desktop and mobile. This unattended task has no persistent user-visible Human tab, so browser injection, full-page screenshots, and direct inspection were the available visual evidence.

## Overall impression

The application communicates one stable task sequence: queue context, compact recipe controls, dominant Base/LUT comparison, then the relevant export decision. The layout remains quiet enough for image comparison while status and recovery actions stay visible.

## Layout assessment

- Desktop uses a 280 px queue rail and a flexible workspace; both comparison panels share available width without horizontal overflow at 1440 px.
- Mobile becomes a single-column flow and preserves controls at least 44 px high instead of compressing the desktop layout.
- The hierarchy follows queue → look/exposure → Base/LUT comparison → export, including at a 600 px-tall desktop viewport.
- Spacing stays on the established 4/8/12/16/24/32/48 scale, and export remains reachable through normal page scrolling.

## What's working

- Empty, ready, mobile, and short-desktop states preserve the same semantic task order.
- The selected-file action is primary for one file; batch export becomes the sole primary action when multiple files are eligible.
- Privacy, undeclared output-profile semantics, disabled corrections, batch progress, and stop behavior appear where they matter.
- Queue selection, busy state, errors, comparison canvases, and recovery actions use text and accessibility semantics rather than color alone.
