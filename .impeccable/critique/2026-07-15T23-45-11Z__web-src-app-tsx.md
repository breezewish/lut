## Anti-patterns verdict

**LLM assessment:** The interface does not read as AI-generated. Its restrained rose/neutral palette, borderless hierarchy, familiar controls, and image-first composition serve the editing task without decorative scaffolding, card grids, glass, gradients, or ornamental motion.

**Deterministic scan:** Both CLI scans returned zero findings: `--scope layout web/src` produced `[]`, and the full `web/src` scan produced `[]`. Browser injection reported compact/single-font product-UI warnings and one transparent-background contrast false positive. Inter is intentionally the only family in this product register. The amber assumption text has 8.61:1 contrast against its actual raised surface; the detector measured it against white instead.

**Visual overlays:** Mutable injection succeeded in headless Chromium for desktop empty, desktop ready, and mobile ready states. No user-visible Human tab is available in this unattended desktop session; screenshots at 1440×960, 390×844, 1024×600, and active batch export supplied the visual fallback.

## Overall impression

This is a focused local color tool with a stable hierarchy: queue context, compact processing controls, dominant comparison, and one export decision. The final opportunity was not cosmetic; it was making the visible recipe and serial Worker queue remain truthful during long exports.

## What's working

- Base and LUT canvases dominate every ready layout while labels and assumptions remain close enough to interpret them.
- Desktop, short-height, and mobile layouts change structure rather than merely shrinking type; controls remain at least 44 px and export stays reachable.
- Privacy, unverified LUT output semantics, failure recovery, queue progress, and stop semantics are visible at the moment users need them.

## Priority issues

- **[P1 · resolved] Populated queue rejected further drops:** This broke the promised repeated drag-and-drop workflow. Drop handling now belongs to the queue container, so empty and populated states share one import boundary.
- **[P1 · resolved] Export recipe could visually change mid-operation:** EV, LUT, selection, or new import could insert preview work between serial batch files while the export still used its captured recipe. Import, selection, EV, and LUT controls now lock until completion or stop-after-current.
- **[P2 · resolved] Same-event duplicates entered twice:** The previous filter compared every incoming file only with the pre-existing queue. Each accepted identity now enters the same Set immediately, so duplicates within one chooser/drop event collapse to one item.

## Persona red flags

**Alex (power user):** No blocking red flag remains. Batch export, direct EV entry, look search, recent looks, and stop-after-current support a fast path. Dedicated keyboard accelerators remain the only optional efficiency gap.

**Jordan (first-timer):** The first action, local-processing reassurance, Base/LUT distinction, and export action are explicit. The undeclared output-profile note uses necessary domain language but explains the actual preview assumption inline.

**Sam (accessibility-dependent):** Native controls and Radix Select remain keyboard-operable, focus-visible, labeled, and at least 44 px. Queue selection, comparison busy state, progress, errors, and canvas image semantics are exposed without relying on color alone.

## Minor observations

- One type family is appropriate for this product surface; adding a display face would reduce trust and consistency.
- Searchable in-app help and expert shortcuts may be useful later, but neither is needed to understand or complete the scoped workflow.

## Questions to consider

- Should output-color semantics become a selectable contract only after each LUT family has verified provenance, rather than adding speculative UI now?
